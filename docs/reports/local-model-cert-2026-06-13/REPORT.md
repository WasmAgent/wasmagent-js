# Local-Model Cert — V3 Report (2026-06-13)

> Real-hardware run of the L4 cert pipeline for every alias in
> `packages/model-local/src/registry.ts`. Output of the
> `desktop-agent-feasibility-plan.md` V3 step.

## TL;DR

- **5/5 aliases now have real sha256 pinned** (was 0/5 placeholders).
- **4/5 alias URLs were broken** in the pre-V3 registry — the pipeline
  caught it on the first real download attempt. Each was replaced with
  a verified canonical repo (or its working community mirror).
- **`qwen3.5-0.8b` was retired** — no such checkpoint published on
  HuggingFace as of 2026-06-13. `qwen2.5-1.5b` was added in its slot.
  This is a deliberate registry change reflecting reality, not a typo
  fix.
- **Cert dimension scoring (tool-call / CodeAgent / bilingual)** was
  NOT run in this batch — V3's scope was supply-chain pinning. Per-arm
  cert scores will be filled in by V2's ablation harness (which
  exercises the same models against richer multi-turn-tool-exec items).

## Bugs the cert run surfaced

The pre-V3 registry promised five GGUF files at canonical
HuggingFace URLs. Real HTTP GETs against those URLs:

| alias (pre-V3) | HF URL status | Resolution |
|---|---|---|
| `qwen3-0.6b` (`q4_k_m`) | 404 — only `Q8_0` quant exists in `Qwen/Qwen3-0.6B-GGUF` | Repointed to `Q8_0`. |
| `gemma-3-1b` (`google/gemma-3-1b-it-GGUF`) | 403 — repo is QAT-only; `q4_k_m` doesn't exist there | Repointed to `ggml-org/gemma-3-1b-it-GGUF` (community mirror that publishes the standard quant). |
| `llama-3.2-1b` (`meta-llama/Llama-3.2-1B-Instruct-GGUF`) | 401 — gated repo, unauthenticated download forbidden | Repointed to `lmstudio-community/Llama-3.2-1B-Instruct-GGUF` (community redistribution within the Llama 3.2 Community License). |
| `qwen3.5-0.8b` (any URL) | 404 across HF/hf-mirror/modelscope — checkpoint never published | **Removed from registry.** `qwen2.5-1.5b` added (Stage-0 ≤2GB winner from the parallel evomerge eval, GSM8K 70.5% Q8_0). |

This is the value the plan ascribed to V3 — broken supply chain
caught BEFORE shipping a cert badge.

## What's NOT in this report

- **Per-model cert dimension scores** (tool-call form / picked /
  semantic, CodeAgent rate, bilingual rate). The cert harness exists at
  `examples/benchmarks/local-model-cert.mjs` and passes self-test, but
  the per-model real-machine pass for those dimensions overlaps almost
  entirely with V2's ablation grid (same models × richer suite). V2
  produces them as a side-effect.
- **node-llama-cpp loading times**. V2 runs against Ollama's
  OpenAI-compat endpoint, not via embedded node-llama-cpp; that's
  deliberate (consumer reproducibility — no native build required).

## Reproduction

```bash
# Build the model-local package once
bun run -F '@wasmagent/model-local' build

# Walk every alias, download via multi-mirror, pin sha256
node examples/benchmarks/cert-all-aliases.mjs --skip-cert

# Or just one alias (incremental writes — safe to interrupt)
node examples/benchmarks/cert-all-aliases.mjs --aliases qwen2.5-1.5b

# Verify a pinned alias against current cache
agentkit model verify qwen2.5-1.5b
```
