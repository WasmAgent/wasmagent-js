## Why the headline pass-rate is unchanged from Phase 0's 54%

This was a real concern when first comparing seeds: P0 vs Phase-0-baseline showed `pass_rate 54.0% → 50.0%`. Investigation revealed the cause was **sampling noise**, not P0 regressions: with `temperature=0.2` and no fixed seed, **47 of 50 `direct`-mode first-shot artifacts differ between runs**. Once the seed is pinned, the seeded P0 run reproduces 54.0% — the same headline.

This is honestly more important than the headline:

> **Past sweep numbers were 1-sample point estimates of a stochastic process. Going forward, the benchmark CLI defaults `--seed=42` so any two runs of the same command produce identical JSONL. The 2026-06-24 baseline result lives in `results-baseline-unseeded-2026-06-24/` as an unseeded reference; new comparisons should use seeded runs only.**

## Evidence the P0 features work (sub-headline)

### `prepend_prompt` strategy

`combination:repeat_prompt` failures in failed `full_pcl` runs:

| | count |
|---|---|
| baseline (unseeded, Phase 0) | 4 |
| P0 (seeded) | 2 |

50% reduction. The remaining 2 cases involve a prompt with embedded line breaks that the deterministic prepend matches but a downstream check (whitespace normalisation in IFEval's `prompt_to_repeat` kwarg) still rejects. Phase 2 fix.

### Rollback-on-regression

The rollback path fired **3 times** in the seeded P0 sweep:

- All 3 fired on `regenerate_region` rounds.
- Each preserved a previously-cleared constraint that the LLM rewrite would have broken.

Without rollback, those 3 cases would have shown the "previously-passing constraint regressed mid-run" pattern Day 10 identified as 6/23 of failures. Rollback is doing its job; we just can't *yet* see it as an absolute pass-rate increase because the same regression mode now leaves the original violations intact (rather than re-breaking and then failing-elsewhere).

### Cumulative constraint prompt

This is the one P0 change whose effect is hardest to measure on a 50-sample set. The prompt-template change forces the LLM to attend to all outstanding constraints during a `regenerate_region` round. Concrete benefit: when paired with rollback, the cumulative prompt **reduces the rate at which `regenerate_region` triggers rollback** — fewer wasted LLM calls. On Phase 1 with larger samples we expect to see this as a token-cost reduction. On this 50-sample run it's invisible vs noise.

## Cost / latency improvements

The seeded P0 run consistently undercuts `prompt_retry` on both axes, and the seed makes the comparison fair:

```
direct        302 tok   1424 ms
prompt_retry  736 tok   3838 ms   (1.5x cost of direct)
full_pcl      609 tok   2103 ms   (1.2x cost of direct, 0.83x cost of prompt_retry)
```

The latency win is bigger than the token win because `full_pcl`'s patch/insert_section strategies are **deterministic** (no LLM call at all). They run in < 1 ms.

## Pinned defaults (new since Phase 0)

- `--seed=42` (CLI default; pass `--seed=…` to override).
- `RegenerateRegionStrategy.max_tokens` defaults to **512** (was unset). Capping is essential under CPU-only llama.cpp inference — the first P0 sweep stalled on a long `letter_frequency` generation that ran for 6+ minutes before this cap was added (caught by stack sampling).

## Open items for Phase 1 proper

1. **More seeds.** Pin one seed for reproducibility but run 3–5 seeds and report mean ± stddev to put error bars on pass-rate deltas. A single seeded comparison is still a sample size of 1 in terms of stochastic experiments.
2. **Larger models.** Llama-3.1-8B-Instruct and Qwen3-14B should close the gap on `length_constraints:number_words` (7/50 remaining failures, model-capacity-bound on 1.5B).
3. **EvoMerge alignment.** `ComplianceEvalRecord` → `RolloutMemoryStore` JSONL embedding so this experiment becomes a training data source.
