## How `full_pcl` succeeds

By strategy, in samples where the round cleared the targeted violation:

| strategy | wins | mechanism |
|---|---|---|
| `regenerate_region` | 16 | LLM rewrite, ~half the tokens of full retry |
| `patch` | 8 | deterministic comma-strip / lowercasing — zero LLM tokens |
| `insert_section` | 4 | prepend `<<title>>` / append keyword trailer |

`patch` is the cheapest fix and the most compelling PCL demonstration: it clears `punctuation:no_comma` and `change_case:english_lowercase` violations without any LLM call. Prompt+Retry burns ~150 tokens per retry and often re-introduces the violation.

## Failure deep-dive

See `FAILURE-ANALYSIS-2026-06-24.md` for the 23 `full_pcl` failures grouped into 4 patterns, with Phase-1 prescriptions for each.

Quick preview:
- **6 cases**: repair regression (round 2 LLM rewrite undoes round 1 patch). **Fixable** via rollback-on-regression in `RepairPlanner`.
- **8 cases**: model-capacity ceilings on `length_constraints:number_words` (Qwen-1.5B can't sustain ≥600 words). **Fixable** by Phase-1 model upgrade.
- **4 cases**: `combination:repeat_prompt` triggers safety refusals. **Fixable** by a deterministic `prepend_prompt` strategy.
- **6 cases**: `keywords:frequency` / `letter_frequency` — improvable but lower-priority.

## Reproducing this report

```bash
# From repo root
bun packages/compliance/benchmarks/ifeval/run.ts \
  --limit=50 \
  --out=packages/compliance/benchmarks/ifeval/results
```

The CLI is **idempotent + resumable** (single-writer file lock prevents concurrent sweeps from racing; existing JSONL is skipped on restart). Wall-clock ~45 s on macOS arm64 with Qwen2.5-1.5B-Q4 GGUF.
